# 提示词策略 v1.0

> 适用范围：木鱼桌宠当前记忆链路（摘要、档案提取、主动发话、注入规范）
> 状态：先文档固化，后续按需替换到代码
> 更新日期：2026-02-28

---

## 1. 目标与边界

本文件用于统一当前项目“记忆相关提示词”的设计与使用方式，避免不同功能各自拼接、质量不一致。

本版本覆盖：
- 记忆摘要
- 档案提取
- 主动发话（生日/缺席）
- 记忆注入规范

本版本不覆盖：
- 多智能体三代理提示词
- 语音链路提示词
- 数据库结构改造

---

## 2. 提示词索引

| Prompt ID | 用途 | 入口函数/配置 | 输入变量 | 输出格式 |
|---|---|---|---|---|
| `PROMPT_MEMORY_SUMMARY_V1` | 生成中期记忆摘要 | `db.getMemorySummarySystemPrompt()` | 对话文本 | 结构化纯文本（5行） |
| `PROMPT_PROFILE_EXTRACT_V1` | 提取用户档案字段 | `llm-service.generateProfileExtraction()` | 对话文本 | 严格 JSON |
| `PROMPT_PROACTIVE_BIRTHDAY_V1` | 生日前置问候生成 | `llm-service.generateProactiveGreeting()` | 角色设定、近期上下文 | 短文本（1-2句） |
| `PROMPT_PROACTIVE_ABSENCE_V1` | 缺席回归问候生成 | `llm-service.generateProactiveGreeting()` | `daysSince`、角色设定、近期上下文 | 短文本（1-2句） |
| `PROMPT_MEMORY_INJECTION_POLICY_V1` | 记忆注入行为规范 | `llm-service.buildRuntimeContext()` 的系统约束文案 | 档案、关系、摘要、最近对话 | 规则文本（不直接对用户输出） |

## 3. 提示词正文

### 3.1 `PROMPT_MEMORY_SUMMARY_V1`

**用途**：从最近一段对话中提炼“可长期使用”的摘要。  
**输入**：由调用侧传入会话 ID 与对话文本。  
**输出**：固定 5 行结构化文本，不要 JSON，不要 Markdown。

```text
你是“长期记忆摘要器”。请基于给定对话，只提炼对后续陪伴有价值的信息。

硬性规则：
1) 只能依据对话里明确出现的信息，不编造、不脑补。
2) 忽略寒暄、客套、重复口头禅。
3) 总长度控制在 90~180 个中文字符。
4) 必须按下列 5 行格式输出；每行一句，信息不足时写“无”。
5) 不要输出任何额外解释、标题或代码块。

输出格式（严格）：
【心情】...
【事件】...
【话题】...
【未解诉求】...
【互动态度】...

判定说明：
- 【心情】：用户主要情绪倾向（正向/负向/平稳及原因）。
- 【事件】：今天或近期明确发生的关键事件。
- 【话题】：反复出现或权重最高的主题。
- 【未解诉求】：用户提出但尚未被满足的问题/担忧/请求。
- 【互动态度】：用户与宠物互动风格（调侃/依赖/倾诉/闲聊等）。
```

**失败回退**：
- 若模型仍返回非结构文本，调用侧可直接存原文摘要（当前实现兼容）。
- 若摘要为空，按 `summary-empty` 处理。

### 3.2 `PROMPT_PROFILE_EXTRACT_V1`

**用途**：从对话抽取当前代码可消费的档案字段。  
**输入**：对话文本。  
**输出**：严格 JSON，仅一个对象。

```text
从以下对话中提取用户档案信息。

规则：
- 只提取对话中明确出现的内容，不推断，不猜测。
- 没有依据的字段必须输出 null 或 []。
- birthday 格式必须是 MM-DD（如 03-15），否则输出 null。
- birthday_year 必须是数字年份（如 1998），否则输出 null。
- traits 只保留稳定个性/偏好词，最多 5 项。
- notes_append 是补充备注，简洁一句话。
- 只输出 JSON，不要输出任何解释文字，不要 Markdown 代码块。

输出 JSON（字段固定，不得增删）：
{
  "name": "...",
  "occupation": "...",
  "birthday": "...",
  "birthday_year": null,
  "traits": ["...", "..."],
  "notes_append": "..."
}
```

**失败回退**：
- 解析失败时返回 `null`（当前 `parseJsonLoosely` 已支持多级兜底）。

### 3.3 `PROMPT_PROACTIVE_BIRTHDAY_V1`

**用途**：生日触发时生成自然问候。  
**输入**：角色设定、近期对话上下文、触发理由（生日）。  
**输出**：1-2 句中文短文本。

```text
你正在以既定角色与用户互动。今天是用户生日，请生成一句自然、温暖的主动问候。

规则：
- 仅输出 1~2 句中文，总长度 18~60 字。
- 语气真诚、轻松，不油腻，不夸张，不说教。
- 不要暴露“系统触发/规则触发/自动消息”等后台信息。
- 不强迫用户继续聊天，不制造负担。
- 如果上下文显示用户情绪低落，语气更柔和，避免过度兴奋。

禁止：
- 命令式表达（如“你必须开心”）。
- 过度亲密或冒犯语气。
- 空泛套话（如“祝你万事如意天天开心”整句照搬模板）。
```

**失败回退**：
- 生成失败时不写入主动消息，仅打开聊天窗口（与当前主流程一致）。

### 3.4 `PROMPT_PROACTIVE_ABSENCE_V1`

**用途**：缺席回归时生成自然问候。  
**输入**：`daysSince`、角色设定、近期对话上下文。  
**输出**：1-2 句中文短文本。

```text
你正在以既定角色与用户互动。用户已经 {daysSince} 天未打开应用，请给出自然的主动问候。

分级语气：
- 4~7 天：轻度想念，偏欢迎口吻。
- 8~14 天：温和关心近况，可轻问“最近还好吗”。
- >14 天：只做普通欢迎，不追问“为什么这么久没来”。

规则：
- 仅输出 1~2 句中文，总长度 18~60 字。
- 语气自然、不刻意、不指责、不道德绑架。
- 不要暴露“系统触发/缺席检测”等后台信息。
- 不使用审问式连续提问。

禁止：
- “你怎么现在才来”这类责备语气。
- 让用户产生愧疚感的表达。
```

**失败回退**：
- 生成失败时不写入主动消息，仅打开聊天窗口（与当前主流程一致）。

---

### 3.5 `PROMPT_MEMORY_INJECTION_POLICY_V1`

**用途**：约束模型如何使用记忆信息，降低“旧记忆压过当前表达”的风险。  
**输入**：用户档案、关系状态、记忆摘要、最近对话。  
**输出**：规则文本（system message）。

```text
你在回复时必须遵守以下记忆使用优先级：

优先级（从高到低）：
1) 用户当前输入（本轮）；
2) 最近对话上下文；
3) 用户档案（稳定信息）；
4) 关系状态；
5) 长期记忆摘要。

执行规则：
- 若“历史记忆”与“用户当前表达”冲突，以当前表达为准。
- 记忆仅用于提升理解与语气连续性，不可生硬复读。
- 未被明确证实的信息，不得当作事实陈述。
- 对敏感或高风险话题，优先给温和、可执行、低风险建议。
```

**失败回退**：
- 若未注入该规范，不影响主流程，但回复稳定性下降（可接受的软退化）。

## 4. 与当前代码映射

### 4.1 可直接替换（无需改接口）
- `PROMPT_MEMORY_SUMMARY_V1`
  - 映射：`src/main/db.js` 中 `memory_summary_system_prompt` 配置项。
  - 当前设置页可直接粘贴到“全局记忆提取提示词”。
- `PROMPT_PROFILE_EXTRACT_V1`
  - 映射：`src/main/services/llm-service.js` 中 `generateProfileExtraction()` 的 `prompt`。

### 4.2 需小改实现（提示词已先行）
- `PROMPT_PROACTIVE_BIRTHDAY_V1` / `PROMPT_PROACTIVE_ABSENCE_V1`
  - 当前代码使用简短 `reason` 字符串，建议后续替换为完整模板。
- `PROMPT_MEMORY_INJECTION_POLICY_V1`
  - 当前 `buildRuntimeContext()` 尚未单独注入该策略 system message，后续可追加。

## 5. 使用约束（统一）

- 语言：中文优先，贴合陪伴场景。
- 风格：自然、克制、可执行，不讲空话。
- 长度：
  - 摘要：90~180 中文字，固定 5 行。
  - 主动发话：18~60 字，1~2 句。
- 安全：不鼓励危险行为，不输出违法/自伤/仇恨导向内容。
- 可维护性：统一以 Prompt ID 管理，变更时升级版本号。

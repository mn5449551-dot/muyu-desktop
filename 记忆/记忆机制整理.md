# 木鱼桌宠 - 记忆机制整理（v1.0 实现版）

> **定位**：本文档描述当前 v1.0 实现的单层滚动摘要系统。完整的三层记忆模型（短期/中期/长期）规划见同目录下的《记忆策略方向.md》。

---

## 一、核心概念

这个项目的记忆机制是一个**基于 LLM 的滚动摘要系统**，目的是让 AI 记住跨会话的长期信息。

---

## 二、当前实现的记忆类型（v1.0）

| 类型 | 存储表 | 说明 |
|------|--------|------|
| **记忆摘要** | `memory_summaries` | 自动从聊天记录中提取的关键信息（对应策略文档的"中期记忆"） |
| **用户档案** | `user_profile` | 用户手动填写的固定信息 |

> v1.0 采用**单层滚动摘要**模式：累积 20 条未摘要消息后自动生成摘要，摘要会永久保留并注入到后续对话上下文中。

---

## 三、记忆摘要机制

### 1. 触发方式

- **自动触发**：每次聊天结束后自动检查（`main.js:869`）
- **手动触发**：用户可在设置页点击"生成记忆摘要"按钮

### 2. 触发参数

```javascript
{
  threshold: 20,      // 累积20条未摘要消息后触发
  maxMessages: 40    // 每次最多处理40条消息
}
```

### 3. 摘要生成流程

```
用户聊天 → 消息存入 chat_messages
     ↓
消息数 >= threshold (20条)
     ↓
调用 LLM.generateSummary()
     ↓
将未摘要消息转为: [user] xxx\n[assistant] yyy
     ↓
使用 memorySummarySystemPrompt 提示词
     ↓
LLM 返回摘要 → 存入 memory_summaries
```

### 4. 默认提示词

> 你是长期记忆提取器。请把对话总结为 3-6 条短要点（总计不超过 120 字），仅保留长期有价值信息：事实、偏好、目标、约束、约定。不要复述寒喧，不要编造，不要输出无关解释。

**出处**：`src/main/db.js:17`

### 5. 自定义提示词

- 用户可在设置中修改（20-1200字）
- 会覆盖默认提示词

## 四、记忆的使用方式

### 1. 上下文构建

**函数**：`buildRuntimeContext()` (`src/main/services/llm-service.js:12`)

```javascript
// 1. 获取角色设定的 system prompt
chatSystemPrompt

// 2. 获取用户档案（如果有）
用户档案：
name: 张三
occupation: 程序员
traits: 内向/喜欢技术
notes: ...

// 3. 获取最近5条记忆摘要
长期记忆摘要（按时间从旧到新）：
#1 摘要内容1
#2 摘要内容2
...

// 4. 拼接最近 N 条对话作为上下文
```

### 2. 注入顺序

```
[system] 角色设定 prompt
[system] 用户档案
[system] 长期记忆摘要（所有摘要按时间顺序排列）
[user] 最近对话...
[user
```

## 五、用户档案

### 字段

| 字段 | 说明 |
|------|------|
| `name` | 姓名 |
| `occupation` | 职业 |
| `traits` | 个性标签（逗号分隔） |
| `notes` | 备注 |

### 管理方式

- 用户在设置页手动填写
- 保存后立即注入到后续所有对话的 system message

## 六、数据表结构

### memory_summaries

```sql
CREATE TABLE memory_summaries (
  id          INTEGER PRIMARY KEY,
  summary     TEXT NOT NULL,        -- 摘要内容
  from_msg_id INTEGER NOT NULL,     -- 覆盖的消息起始ID
  to_msg_id   INTEGER NOT NULL,     -- 覆盖的消息结束ID
  ts          INTEGER NOT NULL,     -- 创建时间戳
  session_id  TEXT DEFAULT 'default'
);
```

### user_profile

```sql
CREATE TABLE user_profile (
  key        TEXT PRIMARY KEY,
  value      TEXT NOT NULL,
  updated_at INTEGER NOT NULL
);
```

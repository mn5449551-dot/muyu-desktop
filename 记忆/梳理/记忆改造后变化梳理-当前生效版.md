# 记忆改造后变化梳理（当前生效版）

> 更新时间：2026-02-28  
> 范围：仅基于当前项目代码里“正在生效”的逻辑，不含未落地方案。

## 1. 本次已生效的记忆相关变化

### 1.1 记忆写入链路
- 用户每次发送消息，会先写入 `chat_messages`。
- 助手回复完成后，异步触发 `runSummaryIfNeeded`（默认阈值 `20` 条未摘要消息，最多取 `40` 条做摘要）。
- 摘要成功后会写入：
  - `memory_summaries`（中期记忆）
  - `summary_history`（摘要过程记录：输入/输出 token、区间等）
- 关系状态会随摘要次数推进：`new -> familiar -> close`。
- 每累计 3 次摘要会触发一次长期档案抽取（姓名/职业/生日/traits/notes）。

### 1.2 记忆召回链路（生成前注入）
- 短期记忆：读取最近对话（`maxContext`）。
- 中期记忆：从最近 8 条摘要里做关键词相关度筛选，取 Top 3 注入。
- 长期记忆：注入用户档案（profile）与关系阶段（character_memory）。
- 同时会注入“记忆使用优先级策略词”（当前输入 > 最近上下文 > 档案 > 关系 > 摘要）。

### 1.3 安全与人设（与记忆协同）
- 输入侧：本地规则风控（关键词/正则）与上下文构建并发执行；命中高风险则直接拒答。
- 输出侧：按句子/分段做风险与风格漂移检测；命中时调用一次改写提示词后再下发。
- 记忆摘要与档案抽取都在回复结束后异步执行，不阻塞当前响应。

### 1.4 设置页能力变化
- 新增“记忆”列表页：按角色会话查看摘要、删除摘要（软删除）。
- 新增“主动发话”总开关（缺席/生日触发）。
- 记忆相关高级提示词改为只读展示（MVP 口径），用户只编辑角色提示词。

## 2. 与你预期的对照（产品视角）

### 2.1 已符合
- 有短期/中期/长期三层记忆，并在生成前注入上下文。
- 有输入安全与输出安全/风格双重校验。
- 关键慢链路（摘要、档案抽取）已放到回复后异步执行。
- 用户可感知并管理中期记忆（查看/删除）。

### 2.2 部分符合（仍有偏差）
- “中期记忆留存当天/近期关键信息”目前主要靠“20条触发+LLM摘要”，不是事件级/重要度级触发。
- 输入安全目前是规则引擎，不是小模型判定；效率高，但语义覆盖有限。
- 长期档案抽取频率偏低（每3次摘要一次），且对已存在字段倾向保守不覆盖，可能更新不及时。
- 中期召回窗口为“最近8条摘要里选3条”，更早但重要的历史信息可能命中不足。

### 2.3 当前实现的边界（你需要知晓）
- 删除摘要是软删除且无前端恢复入口。
- 摘要删除后不会被自动重新生成（游标按历史最大 `to_msg_id` 走），这是为了避免反复重算。
- UI 虽然只读展示高级提示词，但底层仍保留 `memory_summary_system_prompt` 配置项（兼容历史数据）。

## 3. 初步结论（是否符合预期）

- 如果你的预期是“MVP 先跑通可用闭环”：当前整体是符合的。
- 如果你的预期是“记忆精准命中+稳定更新+强语义安全判定”：当前还差一轮增强（尤其是中期触发策略、召回覆盖、输入语义风控）。

## 4. 你可以马上核对的验收清单

1. 连续对话超过 20 条后，设置页“记忆”是否新增摘要。
2. 提问近期话题时，回复是否能引用该摘要中的关键信息。
3. 删除某条摘要后，再问同话题，是否明显减少该记忆影响。
4. 提供生日/职业等信息后，若经过多轮对话，是否写入用户档案并在后续生效。
5. 输入明显高风险内容时，是否直接拒答且不走正常生成。
6. 输出出现“AI腔/人设漂移”时，是否被改写后再展示。

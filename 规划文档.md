# 木鱼桌宠 · 项目规划文档

> 基于现有微信小程序"戳它，然后看它出丑"，迁移并扩展为 Mac/Windows 桌面应用。

---

## 一、产品定位

- 个人单机桌面游戏 / 桌宠
- 无边框透明小窗口，常驻桌面，随时点击
- 支持 AI 对话（角色有"大脑"和记忆）
- 后台可自定义角色（上传图片/音效即可替换）
- 通过 GitHub Releases 分发，用户下载安装包双击即用

---

## 二、核心功能

### 游戏主界面
- 点击角色 → 动画 + 音效 + 粒子 + 浮动文字（从小程序迁移）
- 计数器 + 里程碑徽章
- 角色切换

### AI 对话
- 主动点击"聊天"按钮触发对话窗口
- 每个角色拥有独立系统提示词（人设完全自定义）
- 短期记忆：滚动对话上下文窗口
- 长期记忆：两层结构
  - **用户档案（全局共享）**：姓名、语言偏好等基础事实，所有角色可见
  - **对话记忆（角色独立）**：每个角色只记得与自己的对话历史，定期摘要存入 SQLite
- 对话界面：气泡形式，浮在角色旁边
- 支持多个大模型（OpenAI / Anthropic / 本地 Ollama）

### 窗口行为
- 无边框透明窗口，支持鼠标任意拖动
- 关闭窗口后驻留系统托盘，托盘图标可重新唤起
- 鼠标穿透：透明区域穿透桌面，角色本体可点击
- 多显示器：每次启动默认回到主屏固定位置
- 开机自动启动：用户可在设置中自主开关，默认关闭

### 后台管理
**角色配置**
- 上传角色图片（idle / hit 两张静态图）
  - 图片字段预留 `animationType: 'static' | 'gif' | 'spritesheet'`，当前只实现 static，后续支持动态无需改结构
  - 上传界面预留"动态图"入口（当前置灰 + "即将支持"提示）
- 上传主音效 / rare 音效
- 填写角色名称、浮动文字池、AI 系统提示词（人设）
- 一键替换当前角色

**大模型配置**
- 填入 API Key（使用 `safeStorage` 加密存储，不落明文）
- 选择模型厂商和具体模型，附上获取 API Key 的跳转链接
- 默认厂商：字节豆包（火山引擎），对话用 pro 级模型，记忆摘要固定用 Doubao Lite 小模型
- 用户档案编辑（姓名等全局记忆字段）

## 三、技术选型

| 层级 | 技术 | 理由 |
|------|------|------|
| 桌面壳 | **Electron**（锁定 v28/v30 稳定版） | JS 生态，现有代码可直接复用 |
| 前端 UI | **React + Vite** | 游戏 UI + 管理后台统一，组件化便于维护 |
| 本地存储 | **SQLite**（better-sqlite3） | 角色配置、对话历史、记忆摘要 |
| AI 接入 | 直接 HTTP 调用 LLM API | 灵活支持多厂商，从 Electron 主进程发起 |
| 打包 | electron-builder | 输出 .dmg（Mac）/ .exe（Windows） |

### 鼠标穿透实现

```js
// 透明区域穿透，角色本体仍可点击（Mac + Windows 均支持）
win.setIgnoreMouseEvents(true, { forward: true })

// 前端监听 mousemove，判断是否悬浮在角色上，动态切换
// ⚠️ 必须加 requestAnimationFrame 节流，mousemove 原始频率过高会导致 IPC 拥堵和性能抖动
let rafPending = false
document.addEventListener('mousemove', (e) => {
  if (rafPending) return
  rafPending = true
  requestAnimationFrame(() => {
    const onCharacter = isOverCharacter(e.clientX, e.clientY)
    ipcRenderer.send('set-ignore-mouse', !onCharacter)
    rafPending = false
  })
})
```

> ⚠️ 注意：Electron v38+ nightly 版本透明穿透有兼容性问题，开发时锁定稳定版本。

### Electron 进程安全边界

所有窗口必须启用 `contextIsolation`，渲染进程通过 preload 脚本暴露白名单 API，禁止直接 `require`。

```js
// main.js — 创建窗口时
new BrowserWindow({
  webPreferences: {
    contextIsolation: true,      // 必须开启
    nodeIntegration: false,      // 必须关闭
    sandbox: true,
    preload: path.join(__dirname, 'preload.js'),
  }
})

// preload.js — 只暴露白名单 IPC 通道
const { contextBridge, ipcRenderer } = require('electron')
contextBridge.exposeInMainWorld('electronAPI', {
  setIgnoreMouse: (ignore) => ipcRenderer.send('set-ignore-mouse', ignore),
  sendChat: (payload) => ipcRenderer.invoke('llm-chat', payload),
  getCharacters: () => ipcRenderer.invoke('db-get-characters'),
  // ... 按需添加，不暴露任意 ipcRenderer.send
})
```

> IPC 通道名称统一维护在 `src/shared/ipc-channels.js`，主进程只处理白名单内的通道。

### better-sqlite3 打包策略

better-sqlite3 是 native module，打包时需重新编译以匹配 Electron 的 Node 版本，并同时输出 arm64（M 系列）和 x64（Intel）产物。

```json
// electron-builder 配置（package.json 内）
{
  "build": {
    "mac": {
      "target": "dmg",
      "arch": ["arm64", "x64"]
    },
    "afterPack": "./scripts/rebuild-natives.js"
  },
  "scripts": {
    "postinstall": "electron-rebuild -f -w better-sqlite3"
  }
}
```

## 四、数据存储位置

```
app.getPath('userData')/          ← macOS: ~/Library/Application Support/muyu-desktop/
├── muyu.db                       ← SQLite（角色配置、对话历史、记忆摘要、加密 API Key）
└── assets/                       ← 用户上传的角色图片 / 音效
```

> ⚠️ 不使用 `~/Documents/muyu/`：该路径在 iCloud Drive 开启时会被同步，SQLite 并发写入可能导致数据库损坏。

### API Key 存储策略

API Key **不得**明文写入任何文件。使用 Electron 内置的 `safeStorage` API（无需额外依赖），调用系统钥匙串（macOS Keychain / Windows Credential Manager）加密后存入 SQLite。

```js
// 写入
const encrypted = safeStorage.encryptString(apiKey)
db.prepare('UPDATE llm_config SET api_key_enc = ? WHERE id = 1').run(encrypted)

// 读取
const enc = db.prepare('SELECT api_key_enc FROM llm_config WHERE id = 1').get()
const apiKey = safeStorage.decryptString(enc.api_key_enc)
```

## 八、本次规划说明与注意事项

### 规划背景

本文档整理自 2026-02-27 与 Claude Code 的对话。目标是将现有微信小程序（`muyu` 项目）扩展为功能更完整的跨平台桌面应用，同时保留小程序版本独立迭代。两个项目使用独立 Repo 管理：

- `muyu/` — 小程序原版，继续正常维护上线
- `muyu-desktop/` — 桌面版，从零开始重写


### 开发接手注意事项

1. **Electron 版本锁定**：使用 v28 或 v30 稳定版，v38+ nightly 有透明窗口穿透兼容问题
2. **鼠标穿透**：使用 `setIgnoreMouseEvents(true, { forward: true })` + 前端 mousemove 动态切换；必须加 `requestAnimationFrame` 节流，不要全局禁用，不要裸监听 mousemove 直接发 IPC
3. **Electron 安全**：`contextIsolation: true`、`nodeIntegration: false`、`sandbox: true`；渲染进程只能通过 preload 暴露的白名单 API 与主进程通信；IPC 通道名统一在 `src/shared/ipc-channels.js` 维护
4. **数据目录**：固定用 `app.getPath('userData')`，不要写 `~/Documents/` 或其他路径（iCloud 同步 + SQLite 并发写会损坏数据库）
5. **API Key 安全**：一律用 `safeStorage` 加密后存 SQLite，禁止明文写入任何文件
6. **better-sqlite3 打包**：native module，必须配置 `electron-rebuild` 并在 electron-builder 中指定 `arch: ['arm64', 'x64']`，在 CI 或本地打包前先跑 rebuild
7. **动画预留**：角色数据表中加 `animationType` 字段（`'static' | 'gif' | 'spritesheet'`），当前只实现 static，上传 UI 预留动态入口但置灰
8. **记忆架构**：两层——用户档案（全局 SQLite 表，所有角色共享）+ 对话记忆（每角色独立，定期摘要）；摘要触发条件：未摘要消息 ≥ 20 条，或关闭时 ≥ 10 条
9. **assets 图片格式**：现有图片为 `.webp`，桌面端 Electron/Chromium 原生支持，无需转换
10. **角色数量**：当前只上线白虎、木鱼、橙色仓鼠三个角色，其余代码保留但不展示；无解锁机制，三个角色默认全部可用
11. **前端框架**：React + Vite
12. **默认 LLM**：字节豆包（火山引擎），OpenAI 兼容格式，对话用 pro 模型，摘要用 Doubao Lite
13. **分发**：GitHub Releases，无需苹果签名，用户手动在隐私设置中允许即可；更新机制后续版本再加
14. **里程碑徽章**：暂不做
